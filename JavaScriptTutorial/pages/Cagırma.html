JavaScript Fonksiyon Çağırma

Bir JavaScript işlevinin içindeki kod, "bir şey" çağırdığında çalıştırılır.

JavaScript İşlevini Çağırma

İşlev tanımlandığında işlev içindeki kod yürütülmez.

Bir işlevin içindeki kod, işlev çağrıldığında yürütülür.

"Bir işlevi çağırmak" yerine "bir işlevi çağır" terimini kullanmak yaygındır.

"Bir işlevi çağırmak", "bir işlevi başlatmak" veya "bir işlevi yürütmek" demek yaygındır.

Bir Fonksiyonu Fonksiyon Olarak Çağırma

function myFunction (a, b) {

return a * b;

}

myFunction (10, 2); // 20 dönecek

Yukarıdaki işlev herhangi bir nesneye ait değildir. Ancak JavaScript'te her zaman varsayılan bir global nesne vardır.

HTML'de varsayılan global nesne HTML sayfasının kendisidir, bu nedenle yukarıdaki işlev HTML sayfasına "aittir".

Bir tarayıcıda sayfa nesnesi tarayıcı penceresidir. Yukarıdaki işlev otomatik olarak bir pencere işlevi haline gelir.

myFunction () ve window.myFunction () aynı işlevdir:

function myFunction (a, b) {

return a * b;

}

window.myFunction (10, 2); // Ayrıca 20 döndürür

Bu, bir JavaScript işlevini çağırmanın yaygın bir yoludur, ama çok iyi bir uygulama değil.

Genel değişkenler, yöntemler veya işlevler, genel nesnede kolayca ad çakışmaları ve hatalar oluşturabilir.

This Anahtar Kelimesi

JavaScript olarak adlandırılan şey, geçerli koda "sahip" olan nesnedir.

Bunun değeri, bir işlevde kullanıldığında, işlevin "sahibi" olan nesnedir.

Bunun bir değişken olmadığını unutmayın. Bir anahtar kelime. Bunun değerini değiştiremezsiniz.

İpucu: Bu anahtar kelime hakkında daha fazla bilgiyi JS bu Anahtar Kelimede okuyun.

Küresel Nesne

Bir işlev sahip nesnesi olmadan çağrıldığında, bunun değeri genel nesne haline gelir.

Bir web tarayıcısında genel nesne tarayıcı penceresidir.

Bu örnek, pencere nesnesini bunun değeri olarak döndürür:

var x = myFunction (); // x pencere nesnesi olacak

function myFunction () {

return this;

}

Bir işlevi genel işlev olarak çağırmak, bunun değerinin genel nesne olmasına neden olur.

Window nesnesini değişken olarak kullanmak, programınızı kolayca kilitleyebilir.

Bir Fonksiyonu Yöntem Olarak Çağırma

JavaScript'te işlevleri nesne yöntemleri olarak tanımlayabilirsiniz.

Aşağıdaki örnek, iki özellik (firstName ve lastName) ve bir yöntem (fullName) içeren bir nesne (myObject) oluşturur:

var myObject = {firstName: "John",

lastName: "Doe",

fullName: function () {

return this.firstName + "" + this.lastName;

}

}

myObject.fullName (); // "John Doe" geri dönecek

FullName yöntemi bir işlevdir. İşlev nesneye aittir. myObject, işlevin sahibidir.

Buna denilen şey, JavaScript kodunun "sahibi" olan nesnedir. Bu durumda bunun değeri myObject olur.

Dene! Bunun değerini döndürmek için fullName yöntemini değiştirin:

var myObject = {

firstName: "John",

lastName: "Doe",

fullName: function () {

return this;

}

}

myObject.fullName (); // [object Object] (sahip olan nesne) döndürür

Bir işlevi nesne yöntemi olarak çağırmak, bunun değerinin nesnenin kendisi olmasına neden olur.

İşlev Oluşturucu ile İşlev Çağırma

Bir işlev çağırma işleminden önce yeni anahtar sözcük gelirse, bu bir yapıcı çağırma işlemidir.

Yeni bir işlev oluşturduğunuz anlaşılıyor, ancak JavaScript işlevleri nesneler olduğu için aslında yeni bir nesne oluşturuyorsunuz:

// Bu bir işlev yapıcıdır:

function myFunction (arg1, arg2) {

this.firstName = arg1;

this.lastName = arg2;

}


// Bu yeni bir nesne oluşturur

var x = new myFunction ("John", "Doe");

x.firstName; // "John" geri dönecek

Bir yapıcı çağırma yeni bir nesne oluşturur.

Yeni nesne özellikleri ve yöntemleri yapıcısından devralır.

Yapıcıdaki this anahtar kelimesinin bir değeri yoktur.

This anahtar kelimesinin değeri, işlev çağrıldığında oluşturulan yeni nesne olacaktır.
